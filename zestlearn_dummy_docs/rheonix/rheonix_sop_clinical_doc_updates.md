# Rheonix Life Sciences GmbH — SOP for Clinical Documentation Updates

## Purpose
Define the process for preparing and updating cross-functional clinical and product documentation.

## Scope
Applies to Clinical Operations, Product Development, Regulatory Affairs, Project Management, and Data Management.

## Current Workflow
1. Project manager requests status updates from teams.
2. Teams send updates through email or spreadsheet comments.
3. Clinical operations consolidates updates manually.
4. Regulatory and product teams review the compiled document.
5. Final version is shared in project folder.

## Current Pain Points
- Status information is spread across systems and email chains
- Updates are manually consolidated
- Meeting outcomes and action items are easy to miss
- Teams duplicate work when documentation is not visible

## Improvement Opportunity
An AI workflow assistant could summarize updates, extract action items, and create cleaner status overviews.
