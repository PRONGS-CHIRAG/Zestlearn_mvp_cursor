# Rheonix Life Sciences GmbH — Process Description: Project Coordination Across Functions

## Objective
Describe how project status and documentation are coordinated across teams.

## Workflow
1. Project manager requests updates before weekly sync.
2. Clinical, product, and regulatory teams submit updates separately.
3. PMO consolidates updates in spreadsheet and slide deck.
4. Open issues are discussed in meeting.
5. New action items are distributed by email.

## Issues Observed
- No single reliable status summary exists
- Action items are distributed across meeting notes and inboxes
- Cross-functional visibility is limited
- Teams sometimes repeat already completed work

## Desired Future State
- Better visibility into current project state
- Centralized summary of changes and action items
- Less manual coordination effort
